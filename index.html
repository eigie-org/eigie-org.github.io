pip is not defined

2015-12-06
技术
tech python windows pip

图片原作者

更新了Anaconda后之前的一些包没有了。比如：chardet与pyquery。而python 3.4之后是自带pip的，所以可以直接用pip安装。由于之前没有真正用pip安装过，所以安装过程是一波又三折。

#pip 无法运行

按照官方文档，看起来蛮容易的：

$ pip install SomePackage            # latest version
$ pip install SomePackage==1.0.4     # specific version
$ pip install 'SomePackage>=1.0.4'   # minimum version

但是却出现了：

和

这个是由于在python的交互console去执行pip导致的。。。
正确的执行方法是在windows的cmd下：

python -m pip install pyquery

或者是将”Anaconda3\Scripts”也加入环境变量，然后就可以在cmd下直接执行:

pip install pyquery

#pip 无法连接到PiPy导致安装失败

这个原因嘛多半是因为GFW。要解决可以找找看有没有其它地方有这个包，比如pyquery在github上有。
因此可以用下面的办法安装：

pip install -e git+https://github.com/gawel/pyquery.git#egg=pyquery

当然也可以打包下到local后再用pip安装:

pip install -e path/to/project       # project in another directory

bash tips

2014-01-01
技术
linux bash 开机

#bash tips

##添加新环境变量 export
export GOPATH=/Users/me/go

多个环境变量 用『:』分隔

PATH=$PATH:$GOBIN

##添加到开机自动执行的脚本

  • /etc/profile: 此文件为系统的每个用户设置环境信息,当用户第一次登录时,该文件被执行. 并从/etc/profile.d目录的配置文件中搜集shell的设置.

    • /etc/bashrc:为每一个运行bash shell的用户执行此文件.当bash shell被打开时,该文件被读取.
    • ~/.bash_profile:每个用户都可使用该文件输入专用于自己使用的shell信息,当用户登录时,该文件仅仅执行一次!默认情况下,他设置一些环境变量,执行用户的.bashrc文件.
    • ~/.bashrc:该文件包含专用于你的bash shell的bash信息,当登录时以及每次打开新的shell时,该文件被读取.

    • ~/.bash_logout:当每次退出系统(退出bash shell)时,执行该文件.

    • 另外,/etc/profile中设定的变量(全局)的可以作用于任何用户,而/.bashrc等中设定的变量(局部)只能继承/etc/profile中的变量,他们是”父子”关系.

    • ~/.bash_profile 是交互式、login 方式进入 bash 运行的。

    • ~/.bashrc 是交互式 non-login 方式进入 bash 运行的。通常二者设置大致相同，所以通常前者会调用后者
